
## これは何？

筆者が最低限便利に使えると考えるObsidianの環境を設定し、サンプルのノートやテンプレートを入れたものです。

- なんとなくの使い勝手を把握する
- テンプレートやBase機能の使い方をつかむ

ことを目的としています。

## 何から始めたらいいの？

Obsidianは2カラムで表示することができるので、まずこのREADMEのタブを画面の右側に表示するようにします。
README以外に1つ以上のタブが開かれていれば、READMEを右側に持っていくことができます。

## デイリーノート

このファイルをObsidianで開いているなら、既に立ち上げた時点で==デイリーノート==  (日付 の名前のファイル)が作成・表示されていたと思います。
そのタブを開いてみましょう。

タブを閉じていた場合は、左側の **Open today's daily note** か、そのちょっと右上にある **Calendar**から今日の日付をクリックすると、開くことができます。
開いたら、左側のカラムにデイリーノートを表示します。

![[Pasted image 20251202182543.png|300]]
![[Pasted image 20251202182742.png|300]]

開いたデイリーノートにこんなの↓が見えていたら、`alt + r` (altキー押しながらr)と入力してみてください。
![[Pasted image 20251219180354.png]]
これはファイル名(日付)からノートの内容を自動生成するためのテンプレートになっています。
Obsidian起動時にその日のデイリーノートを自動的に追加してテンプレートを適用、表示する設定にしていますが、はじめてObsidianを起動した際にはテンプレートを扱うプラグインが有効になっていないため、このような表示が残ってしまうことがあります。

### リンクの追加

例えば、今日は昨日やっていた作業([[PROJECT_A-200]])の続きをやるとします。
その場合、「メモ」セクションのなかで `[[` と入力します。
すると、既存ノートの候補が現れますので、[[PROJECT_A-200]]を選択すると、既存ノートにリンクされます。
![[Pasted image 20251202183844.png|400]]

リンクはクリックすれば開くことができますし、Ctrlキーを押しながらマウスカーソルをあわせることでも内容をプレビューすることができます。

## 新規ファイルの作成

次に、プロジェクトAの新しい作業 `PROJECT_A-300` を開始するとしましょう。
このときも、「メモ」セクションのなかで `[[` と入力します。
今回は新規作業でノートがないため候補は表示されませんが、そのまま `[[PROJECT_A-300]]` と入力します。
ここで、薄い色のリンクが作られますのでクリックすると、リンクの名前で新規にノートが作られます。
(`Ctrl + n` でリンクとは無関係にファイルを作成することもできます。)

### テンプレートの適用

ファイルができたら、プロジェクトA用のテンプレートをノートに適用します。
ノートを開いた状態で`Alt + e` として**プロジェクトAチケット**を選択すると、テンプレートが適用されます。
![[Pasted image 20251202185345.png]]

プロジェクトA用のテンプレートでは、以下が自動挿入されるようになっています。
(テンプレート自体は `04_templates` に入っていて、自由に書き換え可能です)

- プロパティ(ファイル先頭のメタ情報。フロントマター)
	- ノートのタグ情報
		- JIRA と プロジェクトA
	- チケットへのリンク
		- ファイル名を使ったURLを生成するようにしている
	- チケットのサマリ
		- 作業内容の簡単な説明を入れる想定
- References
	- プロジェクトAの作業でよくあるインプット・アウトプットへのリンク
- メモ
	- 日付(ノート作成日)
		- 作業記録を残す想定
		- 2日目以後は自分で追加する
- タスク
	- タスクが発生したら期限と一緒に記載する

### タスクの記入

たとえば、[[PROJECT_A-300]]をしているときに、先週あった出張の旅費精算が必要だったことを思い出したとします。
そのときには、[[PROJECT_A-300]]を開いたまま `- [ ]` (箇条書きに[ ] をつけたもの、チェックボックス)と入力すると、タスクを記入することができます。

このとき、表示されるメニューから必ず **due date** (締め切り)も一緒に入れるようにします。
今回は、今日の日付を設定しておきます。

## デイリーノート(2)

### タスク

デイリーノートに戻ると、「タスク」内の「期限間近」のところに、先ほど別のノートに設定したタスクが表示されているはずです。
この項目には**すべてのノートに書かれたタスクのうち、締め切りが3日後より前**のものが表示されるようになっています。
なので、締め切りさえ設定しておけば、タスク自体はどのノートに書いてもよく、デイリーノートから漏れなく確認することができます。
(作業中のノートに思いついたときに書いて、そのまま忘れてよいので便利です)

また、「実施予定」のところには、**scheduled date (実施予定日) が翌日以前** のタスクが表示されるようになっています。
予定を早めに設定しておけば、これを見ての前倒し対応も可能です。

タスクについては他にも様々な設定が可能なので、いろいろと触ってみてください。

## base

baseは、ノートをデータベースのように扱うための機能です。
このサンプルでは、`06_bases`に

- プロパティのタグに `TIPS` を含むノート
- プロパティのタグに `プロジェクトA` を含むノート

を表示するためのbaseを置いています。


## ファイルの置き場はどうするの？

Obsidianでは、基本的にノートのフォルダ分けは不要です。
デイリーノートだけは `02_daily` に入れるのが扱いやすいと思いますが、それ以外の様々なメモや記録などはすべて `01_inbox` に入れてしまっても困ることはありません。

ファイルの情報は、以下の方法でアクセスできます。

- 他のノートからのリンク
- baseファイル
- 検索 (`Ctrl + Shift + f)
- デイリーノートの「ナレッジログ」
	- 最終更新日がデイリーノートの日付になっているファイルをリスト表示するようにしています
	- 「確かこの頃に触ったような・・・」をイメージしています

このベースでは、このようなフォルダ分けを行うことを想定しています。
(基本的に内容別のフォルダ分けは行いません)

- `01_index`：デイリーノートと振り返り(FocusSheet)以外のノートすべての格納先
- `02_daily`：デイリーノートの格納先
- `03_image`：画像ファイルの格納先
- `04_templates`：各種テンプレートファイルの格納先
- `05_fovus_sheets`：振り返り(後述)用ファイルの格納先
- `06_bases`：baseファイルの格納先
- `10_materials`：pdfファイルなどの格納先

## 振り返り

筆者は週に1度(よく忘れますが)、日曜日に**週の振り返り**と**次の週の課題設定**をすることにしています。
その際に使っているのが、05_focus_sheets内にある日付のノートです。
(デイリーノートとファイル名が被っていますが、別フォルダに入っているので特に問題ありません。)

毎週日曜日に翌週日曜日の日付のノートを作って、その日に振り返りと次の週のノートを作って・・・という運用にしています。

### Focus sheet

このノートはこのような内容を含んでいます。

#### 10年後の目標

10年後に達成していたいことを書きます。
10年はとても長いので、今の自分では無理だと思えることを書いてしまっても(むしろその方が)よいです。

もちろん目標自体は変えていってもいいですが、毎週変わるものでもないので、テンプレート(`04_templates/FocusSheet_template`)に書いておくのがいいでしょう。

#### 【課題設定時に記入】 集中課題

この1週間に取り組みたいことを書きます。
いくつ書いてもいいですが、多く書いても結局何も達成できないといったことになりがちですし、1週間しかないので多くても5つくらいにしておくのがよいと思います。

表の内容についてはこのようにしています。

- No.
	- 項目番号。1-5くらいまでにしておきます。
- 達成
	- **行動**の達成状況を自己評価したもの。
- 目標の障害
	- **10年後の目標**が今実現できていない言い訳(障害)を書きます。
	- 数年かけて解消していくような、非常に大きな課題・問題を書くことになるはずです。
- 中間目標
	- 目標の障害を乗り越えるために必要となる、中間目標(達成しておきたい状態)を書きます。
	- 数か月～1年程度で達成できそうな目標を書いておきます。
	- イメージとしては、**年間(年度)計画に書く個別目標くらいの粒度**です。
- 行動
	- 中間目標を達成するための、具体的なアクションを書きます。
		- 後述する[[#【振り返り時に記入】 できたこと (何を達成したか)]]の「次にやること」を転記します。
	- 振り返りの頻度(著者の場合は1週間)に応じて、行動の規模は変わってきます。
	- 必ず**自己評価可能な行動**(やったかどうかがYES/NOで判定できる)を書くようにしましょう。
- メリット(目的・期待)
	- 行動を行うにあたっての期待値やその目的を書きます。
	- 行動の結果を評価する際のベースになります。

#### 【振り返り時に記入】 できたこと (何を達成したか)

集中課題の行動について、1週間の振り返りを行います。

- Y：「やったこと」。実施した(実施しなかった)行動について、具体的に書きます。
- W：「わかったこと」。課題設定時のメリット(目的・期待)について、行動によってわかったことを具体的に書きます。
- T：「次にやること」。次に実施する行動について、具体的に書きます。
- M：「メリット(目的・期待)」。「次にやること」によって、何が得られることを期待しているのか、あるいは目的を具体的に書きます。
